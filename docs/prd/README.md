## PRD Template

All PRDs should include:

1. **Overview** — What problem does this solve?
2. **Use Cases** — Who uses this and why?
3. **Architecture** — High-level system design
4. **Implementation Plan** — Phased approach with deliverables
5. **Security Considerations** — Authentication, authorization, data protection
6. **Open Questions** — Decisions pending
7. **Success Metrics** — How do we know it works?

## Contributing

When creating a new PRD:
1. Copy an existing PRD structure
2. Name file descriptively: `{feature-name}.md`
3. Update this index
4. Commit with message: `docs(prd): Add {feature} PRD`
