Preliminary views on today's top image generation models

We ran 9 models across 3 providers through a storybook illustration benchmark. The results changed which models we ship with — and introduced us to Nano Banana 2.

StoryBook Studio generates illustrations across 34 art styles for children's books. Every model we use has to produce high-quality storybook-style images, handle character prompts reliably, and do it fast enough that a parent doesn't lose interest during generation. We recently ran a structured benchmark across every model we have API access to. This is what we found.

The models we tested

Nine models across three providers. Each model received the same two prompts: a character-interaction scene (Michaelangelo the Ninja Turtle standing next to Cinderella) and an environment scene (a cozy cabin in the woods with falling snow). Both in storybook illustration style.

Reliability: who actually returns an image

Before we talk about quality, we need to talk about whether the model returns anything at all. Of 18 total generations, 13 passed and 5 failed. The failure pattern was revealing.

The pattern is immediate: every BFL-hosted Flux model failed on Prompt 1 (the character interaction). All of them succeeded on Prompt 2 (the environment scene). This suggests either a content-filtering issue with the character IP prompt or an API-level reliability problem under certain prompt structures. Either way, for production storybook generation involving named characters, the BFL API is not reliable enough to be a primary provider today.

The Fireworks-hosted Flux models (schnell and dev) both passed 100%. Same underlying model family, different hosting — and completely different reliability outcomes. Provider matters as much as the model.

Speed: the production constraint

In a children's book generator, each page requires an image. A 12-page book means 12 sequential generations plus a cover. If each takes 20 seconds, that's 4 minutes of waiting. Parents leave. Speed is not optional.

Flux Schnell via Fireworks is the fastest at 2.2 seconds average. This is exceptional for production use. Flux Dev FP8 runs at 3.9 seconds with visibly higher quality — our current sweet spot for the cost-speed trade.

Nano Banana (original) averages 5.7 seconds. That is competitive. Nano Banana 2 doubles that to 13.1 seconds — meaningfully slower, but the quality improvement is visible.

Nano Banana 2: the new entrant

Google's Nano Banana family is their native image generation through the Gemini API. We tested three tiers: nano-banana (base), nano-banana-2 (enhanced), and nano-banana-pro (highest tier).

Nano Banana 2 stands out. At 13.1 seconds, it is not the fastest model. But it has three properties that matter for storybook illustration specifically:

  • 100% reliability — both prompts passed, including the character-interaction prompt that tripped every BFL model.
  • High output resolution — 2342KB and 2110KB file sizes indicate high-detail generation. This is the largest output of any Gemini model in the test, and competitive with BFL's best.
  • Native integration — Gemini API is a single synchronous call. No polling, no async status checks, no download step. The simplest integration of any model we tested.

Nano Banana Pro, the highest tier, failed one of two generations and timed out at 70 seconds. For production use, the Pro tier is not stable enough yet. Nano Banana 2 hits the reliability floor while still delivering quality output.

The Flux landscape

Flux remains the dominant open-source image model family. But the hosting provider changes the story completely.

Fireworks-hosted Flux (schnell, dev-fp8) is fast, reliable, and simple. Both models returned images for both prompts. Schnell at 1.9-2.4 seconds is the fastest model in the entire benchmark. Dev FP8 at 3.8-3.9 seconds produces noticeably better images while staying under 4 seconds. The file sizes are smaller (565-905KB) indicating efficient but lower-resolution output compared to Gemini models.

BFL-hosted Flux (flux-2-max, pro, klein-9b, klein-4b) failed every character prompt. When they succeeded on the environment prompt, the output quality and resolution were impressive — flux-2-pro produced a 2003KB image, flux-2-klein-9b hit 2103KB. But the async polling architecture (submit → poll → download) adds complexity, and the 50% failure rate on character prompts is disqualifying for production.

What's missing from this benchmark

This is a preliminary view. What we haven't tested yet:

  • GPT-Image-1 (OpenAI) — recently released, native image generation from GPT-4o. Needs API access and testing. Early reports suggest strong prompt adherence and character consistency.
  • Recraft V3 — designed specifically for illustration and design use cases. The style-conditioning approach could be a strong fit for storybook work.
  • Ideogram 3.0 — known for text rendering in images. Less relevant for pure illustration but worth benchmarking for title pages.
  • Stable Diffusion 3.5 — Stability AI's latest. Open weights, self-hostable. If quality is competitive, the economics at scale are significantly better.
  • Character consistency testing — this benchmark tested single images. The real question for StoryBook Studio is whether a model can produce the same character across 12+ pages. That requires a different test harness.
  • Cost per image — not measured here. Gemini and Fireworks have different pricing models. At scale (thousands of books), cost per image becomes the dominant variable.

Current position

As of today, our production stack for StoryBook Studio uses Flux Dev FP8 via Fireworks as the primary model, with Flux Schnell for preview/draft generation. We are adding Nano Banana 2 as a quality tier for cover pages and key illustrations where the 13-second generation time is acceptable.

The BFL direct API is on hold until the character-prompt reliability issue is resolved. Nano Banana Pro is on watch — if Gemini stabilizes the Pro tier, it could move to Tier 1.

We will run this benchmark again when we have API access to GPT-Image-1 and Recraft V3. The full benchmark with interactive scoring and generated images is available as a standalone tool.
